### 2. 服务器环境配置

#### 2.1 安装 PostgreSQL 15

```bash
# 安装 PostgreSQL 仓库
yum install -y https://download.postgresql.org/pub/repos/yum/reporpms/EL-7-x86_64/pgdg-redhat-repo-latest.noarch.rpm

# 禁用已失效的旧版本仓库
yum-config-manager --disable pgdg12 pgdg13

# 安装 PostgreSQL 15
yum install -y postgresql15-server postgresql15

# 初始化数据库
/usr/pgsql-15/bin/postgresql-15-setup initdb

# 启动并设置开机自启
systemctl start postgresql-15
systemctl enable postgresql-15
```

#### 2.2 配置 PostgreSQL

```bash
# 修改监听地址（允许 Docker 访问）
sudo sed -i "s/#listen_addresses = 'localhost'/listen_addresses = '*'/" /var/lib/pgsql/15/data/postgresql.conf

# 添加 Docker 网络访问权限
echo "host    all             all             172.17.0.0/16           md5" | sudo tee -a /var/lib/pgsql/15/data/pg_hba.conf

# 重启 PostgreSQL
sudo systemctl restart postgresql-15
```

#### 2.3 创建数据库和用户

```bash
# 创建数据库
sudo -u postgres psql -c "CREATE DATABASE apple_order_mgr;"

# 创建用户
sudo -u postgres psql -c "CREATE USER apple_user WITH PASSWORD 'AppleOrder2024!';"

# 授权
sudo -u postgres psql -c "GRANT ALL PRIVILEGES ON DATABASE apple_order_mgr TO apple_user;"
sudo -u postgres psql -c "ALTER DATABASE apple_order_mgr OWNER TO apple_user;"
```

### 4. Docker 容器部署

#### 4.1 构建镜像
```bash
cd /var/www/apple-order-mgr
docker build -t apple-order-mgr:latest .
```

#### 4.2 启动容器
```bash
docker run -d \
  --name apple-order-mgr \
  --restart always \
  -p 3001:3000 \
  --add-host=host.docker.internal:host-gateway \
  -e NODE_ENV=production \
  -e PORT=3000 \
  -e DB_HOST=host.docker.internal \
  -e DB_PORT=5432 \
  -e DB_NAME=apple_order_mgr \
  -e DB_USER=apple_user \
  -e DB_PASSWORD=AppleOrder2024! \
  -e JWT_SECRET=AppleOrderJWTSecret2024SecureKey! \
  -e LOG_LEVEL=info \
  -v /var/www/apple-order-mgr/logs:/app/logs \
  -v /var/www/apple-order-mgr/uploads:/app/uploads \
  apple-order-mgr:latest
```

#### 4.3 验证容器运行
```bash
# 查看容器状态
docker ps | grep apple-order-mgr

# 查看日志
docker logs apple-order-mgr

# 测试健康检查
curl http://localhost:3001/api/health
```

### 5. Nginx 配置

#### 5.1 申请 SSL 证书
```bash
certbot certonly --nginx \
  -d apple.godp.me \
  --non-interactive \
  --agree-tos \
  --email admin@godp.me
```

#### 5.2 配置 Nginx

创建 `/etc/nginx/conf.d/apple-order-mgr.conf`：

```nginx
server {
    listen 80;
    server_name apple.godp.me;

    # HTTP 重定向到 HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name apple.godp.me;

    # SSL 证书配置
    ssl_certificate /etc/letsencrypt/live/apple.godp.me/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/apple.godp.me/privkey.pem;

    # SSL 优化配置
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;
    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 10m;

    # 前端静态文件
    location / {
        root /var/www/apple-order-mgr/frontend/dist;
        try_files $uri $uri/ /index.html;

        # 缓存静态资源
        location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
            expires 1y;
            add_header Cache-Control "public, immutable";
        }
    }

    # API 代理到后端
    location /api/ {
        proxy_pass http://localhost:3001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;

        # 超时设置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }

    # 上传文件访问
    location /uploads/ {
        alias /var/www/apple-order-mgr/uploads/;
        expires 1y;
        add_header Cache-Control "public";
    }

    # 日志
    access_log /var/log/nginx/apple-order-mgr-access.log;
    error_log /var/log/nginx/apple-order-mgr-error.log;

    # 客户端上传大小限制
    client_max_body_size 50M;
}
```

#### 5.3 测试并重载 Nginx
```bash
nginx -t
systemctl reload nginx
```

## 🔍 故障排查

### 1. 容器无法启动

**检查容器日志：**
```bash
docker logs apple-order-mgr
```

**常见问题：**
- **端口占用**: 检查 3001 端口是否被占用
  ```bash
  netstat -tlnp | grep :3001
  ```
- **环境变量错误**: 检查 .env 文件配置
- **镜像问题**: 重新构建镜像

### 2. 数据库连接失败

```bash
systemctl status postgresql-15
```

```bash
cat /var/lib/pgsql/15/data/pg_hba.conf | grep 172.17
```

```bash
PGPASSWORD='AppleOrder2024!' psql -h host.docker.internal -U apple_user -d apple_order_mgr
```
