## 一、功能概述

基于 `recipients.tag` 和 `orders.tag` 字段实现渠道管理功能，支持渠道统计、订单查询和渠道名称修改。

### 技术方案

- **不新增数据库表**（方案A）
- 动态聚合统计渠道数据
- 使用事务实现级联更新

### 2.5 修改渠道名称

**接口**: `PUT /api/channels/:tag`

**请求体**:
```json

  "newTag": "新渠道名称"
}
```

**结果**: ✅ 通过（逻辑验证）

**测试场景**:
1. ✅ 原渠道不存在 → 返回 404
