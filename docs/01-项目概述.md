# Apple 订单管理系统 - 项目概述

## 一、项目背景

### 业务场景
用户使用第三方预订助手（NULL AOS Helper）在 Apple 官网批量预订 iPhone 等产品，订单确认邮件会发送到指定邮箱。由于订单量大、涉及多个 Apple ID 和取机人，需要一个统一的管理系统来：
- 自动监听订单邮件
- 提取订单信息并与官网数据同步
- 管理多个 Apple ID 和取机人
- 实时跟踪订单状态

### 核心需求
1. **实时邮件监听**：收到邮件后 10 秒内开始处理
2. **自动数据提取**：从邮件和 Apple 官网提取订单信息
3. **多账号管理**：支持多个 Apple ID 的订单管理
4. **取机人管理**：记录和管理取机人信息
5. **状态同步**：定期从官网更新订单状态
6. **数据展示**：提供订单列表、详情和统计分析

## 三、核心模块

### 3.1 邮件监听服务
- **技术方案**: IMAP IDLE 实时推送
- **监听邮箱**: 单个 QQ/163 邮箱
- **触发时机**: 收到新邮件后立即触发（<1秒）
- **处理目标**: 10 秒内完成解析和存储

### 3.2 邮件解析器
- **支持格式**: NULL AOS Helper 邮件格式
- **核心功能**:
  - 提取 Apple ID
  - 提取订单号和订单链接
  - 解析多商品信息（用 @ 分隔）
  - 提取取机人信息
  - 提取付款方式

### 3.3 订单爬虫
- **目标网站**: Apple 官网订单详情页
- **爬取策略**: axios + cheerio (轻量级 HTTP 请求)
- **反爬措施**: 代理池 + 请求延时
- **数据提取**: JSON 数据解析 + HTML 回退

### 3.4 代理池管理
- **方案**: 第三方代理服务商（快代理/芝麻代理）
- **轮换策略**: 每个请求随机选择代理
- **延时控制**: 每个订单间隔 5-10 秒
- **成本**: 月费几十到几百元

### 3.5 定时任务
- **更新频率**:
  - 未发货订单: 每小时更新
  - 已发货订单: 每 6 小时更新
  - 已完成/已取消: 不再更新
- **任务调度**: node-cron

### 3.6 数据存储
- **数据库**: PostgreSQL
- **核心表**:
  - `apple_ids`: Apple ID 管理
  - `recipients`: 取机人管理
  - `orders`: 订单主表
  - `email_logs`: 邮件处理记录
  - `crawl_logs`: 爬虫日志

## 四、数据流程

### 4.1 新订单处理流程

```
1. 邮件服务器收到新邮件
   ↓ (< 1秒)
2. IMAP IDLE 推送通知
   ↓
3. 拉取邮件内容
   ↓
4. 邮件解析器提取信息
   - Apple ID
   - 订单号、订单链接
   - 多商品列表（型号、名称、数量）
   - 取机人信息（姓名、身份证后四位、标签）
   - 付款方式
   ↓
5. 数据存储
   - 自动创建/匹配 Apple ID
   - 自动创建/匹配取机人
   - 创建订单记录
   ↓ (2-5秒)
6. 触发订单爬虫
   - 使用代理 IP 访问订单链接
   - 提取官网订单详情
   - 更新订单状态、门店信息
   ↓
7. 记录处理日志
   ↓
✅ 总耗时: 5-10 秒
```

### 4.2 订单状态更新流程

```
1. 定时任务触发（每小时）
   ↓
2. 查询所有未完成订单
   ↓
3. 遍历订单列表（使用代理 IP + 延时）
   ↓
4. 访问订单链接，提取最新状态
   ↓
5. 对比状态变化
   - 状态改变 → 更新数据库
   - 状态未变 → 记录日志
   ↓
6. 更新日志表
```

## 六、关键特性

### 6.1 实时性
- IMAP IDLE 保持长连接，邮件到达后 < 1 秒推送
- 收到邮件 10 秒内完成处理和存储

### 6.2 可靠性
- 邮件 UID 去重，防止重复处理
- 爬虫失败自动重试
- 完整的错误日志记录

### 6.3 扩展性
- 支持多 Apple ID 和取机人管理
- 支持多商品订单
- 数据库索引优化查询性能

### 6.4 安全性
- 邮箱授权码加密存储
- HTTPS 传输
- API 鉴权保护
