### 🔧 技术亮点

1. **JWT 无状态认证**
   - 无需服务端存储 session，支持分布式部署
   - Token 包含用户基本信息（userId、username、role）
   - 有效期 7 天，可通过环境变量配置

2. **密码加密**
   - 使用 bcrypt 算法（salt rounds = 10）
   - 密码自动加密 Hook（Sequelize beforeCreate、beforeUpdate）
   - 只存储哈希值，永不存储明文

3. **登录失败锁定**
   - 基于数据库字段实现（failed_login_attempts、locked_until）
   - 连续失败次数累计，达到阈值自动锁定
   - 锁定时间可配置（默认 15 分钟）
   - 成功登录自动重置失败次数

4. **角色权限控制**
   - 中间件工厂函数：requireRole(['admin'])
   - 支持多角色检查：requireRole(['admin', 'manager'])
   - 权限检查失败返回 403 Forbidden

5. **强制修改密码**
   - force_password_change 字段控制
   - 中间件检查，只允许访问修改密码接口
   - 修改密码后自动清除标记

6. **实例方法设计**
   - User 模型包含完整的业务方法
   - comparePassword() - 密码比对
   - incrementFailedAttempts() - 递增失败次数并自动锁定
   - unlockAccount() - 解锁账号并重置失败次数
   - isLocked() - 检查锁定状态

### 🎯 技术决策

#### 1. 为什么选择 JWT 而不是 Session？
- **无状态**：支持分布式部署，无需共享 session 存储
- **性能**：减轻数据库压力，不需要每次请求查询 session
- **简单**：不需要 Redis 等额外存储，降低系统复杂度
- **移动友好**：Token 可存储在 localStorage，适合前后端分离

#### 2. 为什么使用 bcrypt 而不是其他加密算法？
- **安全**：Adaptive hash function，自动加盐，抗彩虹表
- **性能平衡**：salt rounds = 10 在安全性和性能之间取得平衡
- **成熟**：广泛使用，经过时间验证
- **简单**：API 简洁，易于使用

#### 3. 锁定时长为什么选择 15 分钟？
- **安全性**：足够阻止暴力破解
- **用户体验**：不会太长，避免误锁定后长时间无法登录
- **可配置**：通过环境变量 LOCK_DURATION_MINUTES 可调整

#### 4. 为什么失败次数阈值是 5 次？
- **安全性**：足够低，防止暴力破解
- **容错性**：允许用户多次尝试（忘记密码）
- **行业标准**：大多数系统使用 3-5 次

const transaction = await sequelize.transaction();

// orderService.js - saveOrderFromEmail()
// 1. 查找 apple_ids 表
const appleAccount = await AppleId.findOne({ 
  where: { appleId: emailData.appleId }

  // 填充外键 + 快照
  appleData.appleIdRef = appleAccount.id;
  appleData.applePassword = appleAccount.password;

const recipient = await Recipient.findOne({

  }
});
if (recipient) {

  recipientData.recipientRef = recipient.id;
  recipientData.recipientIdCard = recipient.idCardNumber;
  // ... 其他字段
}

// 3. 创建订单（事务）
await Order.create({ ...appleData, ...recipientData, ... }, { transaction });
