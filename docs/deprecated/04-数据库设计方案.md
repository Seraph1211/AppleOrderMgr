# 数据库设计方案

## 一、数据库选型

### PostgreSQL

**选择理由**:
- ✅ 支持 JSONB 数据类型（存储原始邮件和爬虫数据）
- ✅ 强大的索引功能（提高查询性能）
- ✅ 支持并发写入（适合实时邮件处理）
- ✅ 开源免费，社区活跃
- ✅ 适合云服务器部署

## 三、表结构详细设计

### 3.1 apple_ids - Apple ID 管理表

**用途**: 管理多个 Apple ID 账号

```sql
CREATE TABLE apple_ids (
  id SERIAL PRIMARY KEY,
  apple_id VARCHAR(255) UNIQUE NOT NULL,  -- Apple ID 邮箱
  nickname VARCHAR(255),                  -- 备注名称（如：主账号、测试账号）
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

-- 索引
CREATE INDEX idx_apple_ids_apple_id ON apple_ids(apple_id);

-- 注释
COMMENT ON TABLE apple_ids IS 'Apple ID 管理表';
COMMENT ON COLUMN apple_ids.apple_id IS 'Apple ID 邮箱地址，全局唯一';
COMMENT ON COLUMN apple_ids.nickname IS '备注名称，方便用户识别';
```

**示例数据**:
```sql
INSERT INTO apple_ids (apple_id, nickname) VALUES
  ('tzvcantetc8k@hotmail.com', '主账号'),
  ('test123@gmail.com', '测试账号');
```

### 3.2 recipients - 取机人管理表

**用途**: 管理取机人信息

```sql
CREATE TABLE recipients (
  id SERIAL PRIMARY KEY,
  name VARCHAR(100) NOT NULL,             -- 姓名
  id_card_last4 VARCHAR(10),              -- 身份证后四位
  phone VARCHAR(20),                      -- 手机号（可选）
  tag VARCHAR(100),                       -- 标签（如：天津、水果惠）
  notes TEXT,                             -- 备注
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW(),
  
  -- 唯一约束：同名同身份证后四位视为同一人
  CONSTRAINT uk_recipient_identity UNIQUE(name, id_card_last4)
);

-- 索引
CREATE INDEX idx_recipients_name ON recipients(name);
CREATE INDEX idx_recipients_tag ON recipients(tag);

-- 注释
COMMENT ON TABLE recipients IS '取机人管理表';
COMMENT ON COLUMN recipients.name IS '取机人姓名';
COMMENT ON COLUMN recipients.id_card_last4 IS '身份证后四位，用于身份验证';
COMMENT ON COLUMN recipients.tag IS '取机人标签，可能代表地区或批次';
```

**示例数据**:
```sql
INSERT INTO recipients (name, id_card_last4, tag) VALUES
  ('李浩', '603X', '天津'),
  ('冉念', '5904', '水果惠');
```

### 3.3 orders - 订单主表

**用途**: 存储订单核心信息

```sql
CREATE TABLE orders (
  id SERIAL PRIMARY KEY,
  order_number VARCHAR(50) UNIQUE NOT NULL,       -- Apple 订单号（W + 9位数字）
  apple_id_ref INT REFERENCES apple_ids(id) ON DELETE CASCADE,
  recipient_ref INT REFERENCES recipients(id) ON DELETE SET NULL,
  
  -- 产品信息（JSONB 数组，支持多商品）
  products JSONB NOT NULL,                        -- 商品列表
  /*
  [
    {
      "modelId": "MG714CH/A",
      "name": "iPhone 17 鼠尾草绿色 256G",
      "quantity": 2,
      "status": "READY_FOR_PICKUP",
      "imageUrl": "https://..."
    },
    {
      "modelId": "HNPW2ZM/A",
      "name": "Belkin Secure Holder 挂绳",
      "quantity": 1,
      "status": "READY_FOR_PICKUP",
      "imageUrl": "https://..."
    }
  ]
  */
  
  -- 订单状态
  status VARCHAR(50) DEFAULT 'submitted',         -- 订单状态
  /*
  状态值：
  - submitted: 已提交（邮件刚收到）
  - processing: 处理中
  - ready: 准备就绪可取货
  - shipped: 已发货
  - delivered: 已送达
  - cancelled: 已取消
  - pickup_cancelled: 取货已取消
  */
  
  -- 订单信息
  order_url TEXT NOT NULL,                        -- Apple 订单详情链接
  payment_method VARCHAR(50),                     -- 付款方式（支付宝/微信/信用卡）
  
  -- 取货信息
  pickup_store VARCHAR(255),                      -- 取机门店（如：Apple 重庆万象城）
  store_directions_url TEXT,                      -- 门店营业时间与路线 URL
  
  -- 时间信息
  order_date TIMESTAMP,                           -- 预订时间（来自邮件）
  order_placed_date DATE,                         -- 下单日期（来自官网）
  estimated_delivery TIMESTAMP,                   -- 预计送达
  actual_delivery TIMESTAMP,                      -- 实际送达
  
  -- 原始数据
  email_raw TEXT,                                 -- 邮件原文
  crawled_data JSONB,                             -- 从官网爬取的完整 JSON
  
  -- 时间戳
  created_at TIMESTAMP DEFAULT NOW(),
  updated_at TIMESTAMP DEFAULT NOW()
);

-- 索引
CREATE INDEX idx_orders_order_number ON orders(order_number);
CREATE INDEX idx_orders_apple_id_ref ON orders(apple_id_ref);
CREATE INDEX idx_orders_recipient_ref ON orders(recipient_ref);
CREATE INDEX idx_orders_status ON orders(status);
CREATE INDEX idx_orders_order_date ON orders(order_date DESC);

-- JSONB 索引（提高商品查询性能）
CREATE INDEX idx_orders_products_gin ON orders USING GIN(products);

-- 注释
COMMENT ON TABLE orders IS '订单主表';
COMMENT ON COLUMN orders.order_number IS 'Apple 订单号，格式：W + 9位数字';
COMMENT ON COLUMN orders.products IS '商品列表（JSONB 数组），支持多商品';
COMMENT ON COLUMN orders.status IS '订单状态：submitted/processing/ready/shipped/delivered/cancelled/pickup_cancelled';
COMMENT ON COLUMN orders.email_raw IS '邮件原文，用于审计和问题排查';
COMMENT ON COLUMN orders.crawled_data IS '从 Apple 官网爬取的完整 JSON 数据';
```

**示例数据**:
```sql
INSERT INTO orders (
  order_number, apple_id_ref, recipient_ref, 
  products, status, order_url, payment_method, 
  pickup_store, order_date
) VALUES (

  'Apple 重庆万象城',

```

### 3.4 email_logs - 邮件处理日志表

**用途**: 记录邮件处理状态，防止重复处理

```sql
CREATE TABLE email_logs (
  id SERIAL PRIMARY KEY,
  email_uid VARCHAR(255) UNIQUE NOT NULL,  -- 邮件唯一标识（IMAP UID）
  email_subject VARCHAR(500),              -- 邮件主题
  processed BOOLEAN DEFAULT FALSE,         -- 是否已处理
  processed_at TIMESTAMP,                  -- 处理时间
  error_message TEXT,                      -- 错误信息
  created_at TIMESTAMP DEFAULT NOW()
);

-- 索引
CREATE INDEX idx_email_logs_email_uid ON email_logs(email_uid);
CREATE INDEX idx_email_logs_processed ON email_logs(processed);
CREATE INDEX idx_email_logs_created_at ON email_logs(created_at DESC);

-- 注释
COMMENT ON TABLE email_logs IS '邮件处理日志表，用于防止重复处理';
COMMENT ON COLUMN email_logs.email_uid IS '邮件 UID，来自 IMAP 服务器';
COMMENT ON COLUMN email_logs.processed IS '是否已成功处理';
```

### 3.5 crawl_logs - 爬虫日志表

**用途**: 记录爬虫执行日志

```sql
CREATE TABLE crawl_logs (
  id SERIAL PRIMARY KEY,
  order_id INT REFERENCES orders(id) ON DELETE CASCADE,
  proxy_ip VARCHAR(50),                    -- 使用的代理 IP
  success BOOLEAN NOT NULL,                -- 是否成功
  response_time INT,                       -- 响应时间（毫秒）
  http_status INT,                         -- HTTP 状态码
  error_message TEXT,                      -- 错误信息
  created_at TIMESTAMP DEFAULT NOW()
);

-- 索引
CREATE INDEX idx_crawl_logs_order_id ON crawl_logs(order_id);
CREATE INDEX idx_crawl_logs_success ON crawl_logs(success);
CREATE INDEX idx_crawl_logs_created_at ON crawl_logs(created_at DESC);

-- 注释
COMMENT ON TABLE crawl_logs IS '爬虫执行日志表';
COMMENT ON COLUMN crawl_logs.proxy_ip IS '使用的代理 IP，格式：host:port';
COMMENT ON COLUMN crawl_logs.response_time IS '响应时间（毫秒）';
```

## 四、数据关系

### 4.1 一对多关系

```
apple_ids (1) ──────< orders (N)
recipients (1) ─────< orders (N)
orders (1) ─────────< crawl_logs (N)
```

### 4.2 级联删除策略

- **删除 Apple ID**: 级联删除其所有订单（`ON DELETE CASCADE`）
- **删除取机人**: 订单的 recipient_ref 设为 NULL（`ON DELETE SET NULL`）
- **删除订单**: 级联删除其爬虫日志（`ON DELETE CASCADE`）

## 五、查询示例

### 5.1 查询某个 Apple ID 的所有订单

```sql
SELECT 
  o.order_number,
  o.status,
  o.products,
  o.order_date,
  o.pickup_store,
  a.apple_id,
  r.name AS recipient_name
FROM orders o
JOIN apple_ids a ON o.apple_id_ref = a.id
LEFT JOIN recipients r ON o.recipient_ref = r.id
WHERE a.apple_id = 'tzvcantetc8k@hotmail.com'
ORDER BY o.order_date DESC;
```

### 5.2 查询未完成的订单（用于定时更新）

```sql
SELECT 
  id, order_number, order_url, status
FROM orders
WHERE status NOT IN ('delivered', 'cancelled', 'pickup_cancelled')
ORDER BY order_date DESC;
```

### 5.3 统计某个取机人的订单数量

```sql
SELECT 
  r.name,
  r.tag,
  COUNT(o.id) AS order_count,
  SUM((o.products::jsonb -> 0 ->> 'quantity')::int) AS total_quantity
FROM recipients r
LEFT JOIN orders o ON r.id = o.recipient_ref
GROUP BY r.id, r.name, r.tag
ORDER BY order_count DESC;
```

### 5.4 查询包含特定商品的订单

```sql
-- 查询所有包含 iPhone 的订单
SELECT 
  order_number,
  products,
  order_date
FROM orders
WHERE products::text ILIKE '%iPhone%'
ORDER BY order_date DESC;
```

### 5.5 爬虫成功率统计

```sql
SELECT 
  DATE(created_at) AS date,
  COUNT(*) AS total_crawls,
  SUM(CASE WHEN success THEN 1 ELSE 0 END) AS success_count,
  ROUND(AVG(response_time), 2) AS avg_response_time
FROM crawl_logs
WHERE created_at >= NOW() - INTERVAL '7 days'
GROUP BY DATE(created_at)
ORDER BY date DESC;
```

## 六、数据迁移脚本

### 6.1 初始化数据库

```sql
-- 创建数据库
CREATE DATABASE apple_orders
  WITH ENCODING 'UTF8'
  LC_COLLATE='zh_CN.UTF-8'
  LC_CTYPE='zh_CN.UTF-8'
  TEMPLATE=template0;

\c apple_orders;

-- 按顺序创建表
\i schema/01_apple_ids.sql
\i schema/02_recipients.sql
\i schema/03_orders.sql
\i schema/04_email_logs.sql
\i schema/05_crawl_logs.sql
```

### 6.2 创建触发器（自动更新 updated_at）

```sql
-- 创建通用更新触发器函数
CREATE OR REPLACE FUNCTION update_updated_at_column()
RETURNS TRIGGER AS $$
BEGIN
  NEW.updated_at = NOW();
  RETURN NEW;
END;
$$ LANGUAGE plpgsql;

-- 为 orders 表添加触发器
CREATE TRIGGER trigger_orders_updated_at
BEFORE UPDATE ON orders
FOR EACH ROW
EXECUTE FUNCTION update_updated_at_column();

-- 为 apple_ids 表添加触发器
CREATE TRIGGER trigger_apple_ids_updated_at
BEFORE UPDATE ON apple_ids
FOR EACH ROW
EXECUTE FUNCTION update_updated_at_column();

-- 为 recipients 表添加触发器
CREATE TRIGGER trigger_recipients_updated_at
BEFORE UPDATE ON recipients
FOR EACH ROW
EXECUTE FUNCTION update_updated_at_column();
```

## 七、数据备份策略

### 7.1 定时备份

```bash
#!/bin/bash
# 每天凌晨 2 点备份数据库

BACKUP_DIR="/var/backups/apple_orders"
DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_FILE="$BACKUP_DIR/apple_orders_$DATE.sql"

# 创建备份
pg_dump -U postgres -d apple_orders > "$BACKUP_FILE"

# 压缩备份文件
gzip "$BACKUP_FILE"

# 删除 7 天前的备份
find "$BACKUP_DIR" -name "*.sql.gz" -mtime +7 -delete
```

### 7.2 恢复数据

```bash
gunzip -c apple_orders_20260706.sql.gz | psql -U postgres -d apple_orders
```

## 八、性能优化建议

### 8.1 索引优化

- ✅ 为外键字段创建索引
- ✅ 为常用查询条件创建索引（status, order_date）
- ✅ 为 JSONB 字段创建 GIN 索引

### 8.2 查询优化

```sql
-- 使用 EXPLAIN ANALYZE 分析慢查询
EXPLAIN ANALYZE
SELECT * FROM orders WHERE status = 'ready';

-- 添加索引后再次分析
CREATE INDEX idx_orders_status ON orders(status);
EXPLAIN ANALYZE
SELECT * FROM orders WHERE status = 'ready';
```

// Sequelize 配置
