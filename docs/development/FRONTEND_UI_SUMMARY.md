## 🎯 设计决策

### 决策 1: 浅色主题替代深色主题

**背景**: 初始使用 UI/UX Pro Max Skill 生成的是深色主题（OLED）

**用户反馈**: "我对现在的风格不太满意，我不要深色"

**调整**:
- 重新生成浅色主题设计系统
- 主色从青色改为深蓝色
- 背景从深色改为白色/浅灰色
- 文字从浅色改为深色

**结果**: 界面更清晰，符合管理后台的专业感

### 决策 2: 表格列表替代卡片网格

**背景**: Apple ID 和取机人管理页面初始使用卡片网格布局

**用户反馈**: "除了仪表盘中的数据之外，数据不要以卡片的形式展示，以列表的形式展示就行"

**调整**:
- Apple ID 管理页面改为表格列表
- 取机人管理页面改为表格列表
- 保留仪表板的统计卡片

**结果**: 数据密度更高，更适合数据管理

### 决策 3: 统一的组件样式系统

**原因**: 确保后续开发的一致性

**实现**:
- 在 `index.css` 中定义统一的组件类
- 所有页面使用相同的类名
- 避免重复定义样式

**优势**:
- 维护方便
- 风格统一
- 代码简洁
