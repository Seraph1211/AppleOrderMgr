## ⚡ 常用命令

### 容器管理
```bash
# 查看状态
docker ps | grep apple-order-mgr

# 查看日志
docker logs -f apple-order-mgr

# 重启应用
docker restart apple-order-mgr

# 进入容器
docker exec -it apple-order-mgr sh
```

### 应用重新部署
```bash
# 1. 上传代码（本地执行）
rsync -avz --exclude 'node_modules' --exclude '.git' . PP_CloudServer:/var/www/apple-order-mgr/

# 2. 重建并重启（服务器执行）
cd /var/www/apple-order-mgr
docker build -t apple-order-mgr:latest .
docker stop apple-order-mgr && docker rm apple-order-mgr
docker run -d --name apple-order-mgr --restart always -p 3001:3000 \
  --add-host=host.docker.internal:host-gateway \
  -e NODE_ENV=production -e PORT=3000 \
  -e DB_HOST=host.docker.internal -e DB_PORT=5432 \
  -e DB_NAME=apple_order_mgr -e DB_USER=apple_user \
  -e DB_PASSWORD=AppleOrder2024! \
  -e JWT_SECRET=AppleOrderJWTSecret2024SecureKey! \
  -e LOG_LEVEL=info \
  -v /var/www/apple-order-mgr/logs:/app/logs \
  -v /var/www/apple-order-mgr/uploads:/app/uploads \
  apple-order-mgr:latest
```

## 🔥 紧急故障处理

### 应用无响应
```bash
# 1. 检查容器状态
docker ps -a | grep apple-order-mgr

# 2. 查看日志找错误
docker logs --tail 100 apple-order-mgr

# 3. 重启容器
docker restart apple-order-mgr

# 4. 如果还不行，重建容器
docker stop apple-order-mgr && docker rm apple-order-mgr
# 然后用上面的 docker run 命令重新创建
```

### 数据库连接失败
```bash
# 1. 检查 PostgreSQL 状态
systemctl status postgresql-15

# 2. 重启 PostgreSQL
systemctl restart postgresql-15

# 3. 检查网络连接
PGPASSWORD='AppleOrder2024!' psql -h localhost -U apple_user -d apple_order_mgr -c "SELECT 1;"
```

### Nginx 502 错误
```bash
# 1. 检查后端是否运行
curl http://localhost:3001/api/health

# 2. 检查 Nginx 配置
nginx -t

# 3. 重启 Nginx
systemctl restart nginx
```

### 磁盘空间不足
```bash
# 1. 查看磁盘使用
df -h

# 2. 清理 Docker 镜像
docker system prune -a

# 3. 清理日志
find /var/log/nginx -name "*.log" -mtime +30 -delete
find /var/www/apple-order-mgr/logs -name "*.log" -mtime +30 -delete
```
